# CP-SAT 光伏生产车间订单排产系统

本项目面向光伏生产车间的多订单、多产线排产场景，基于 Google OR-Tools 的 CP-SAT 约束规划求解器构建生产排产模型。系统以 Excel 文件作为输入和输出载体，支持普通订单排产、停电影响下的产能调整排产，以及插单后的动态重排。

项目重点解决以下问题：在有限产线和有限生产周期内，根据订单需求量、最早开工日期、最晚完工日期、停电计划和插单扰动等信息，自动生成满足主要生产约束且尽量减少换线、延期和原计划扰动的排产方案。

---

## 一、项目功能

### 1. 普通订单排产

系统根据订单的需求量、最早开工日期和最晚完工日期自动生成排产计划，支持多个订单在多条产线上的分配。模型会在满足订单生产需求的前提下，尽量减少换线次数、订单生产跨度、产线负载波动和超产量。

### 2. 停电影响排产

系统支持读取独立的停电计划 Excel 文件，并根据停电信息动态调整不同日期、不同产线的可用产能。停电排产支持以下情况：

- 全厂停电；
- 部分产线停电；
- 停电前一天产能下降；
- 停电期间受影响产线产能为 0；
- 停电结束后产能逐步恢复；
- 当停电影响导致当前月份无法完成订单时，系统可启用软交期并自动跨月扩展求解。

### 3. 插单重排

系统支持在已有排产结果的基础上进行插单重排。程序会读取 `插单输入` sheet，并自动判断是否启用插单模式。插单场景中，系统会读取旧排产结果，冻结插单日期之前已经执行或已经下发的计划，并在满足插单需求的同时尽量减少对原计划的扰动。

插单类型包括：

- 全新插单：原订单中不存在该订单名称；
- 原订单加量：原订单中存在该订单，且旧计划中该订单在插单日期后仍有生产安排；
- 同名新批次：原订单中存在相同名称，但旧计划中该订单已经在插单日前完成。

### 4. Excel 结果导出

系统会将排产结果导出为 Excel 文件，方便生产计划人员查看。输出内容包括订单视图、按月份拆分的产线排产图，以及订单每日产量明细。

默认输出文件如下：

| 排产类型 | 默认输出文件 |
|---|---|
| 普通排产 / 停电排产 | `CP_SAT_排产结果.xlsx` |
| 插单重排 | `CP_SAT_插单排产结果.xlsx` |

## 四、输入文件说明

### 1. 订单输入文件：`input_orders.xlsx`

订单输入文件至少需要包含 `订单输入` sheet。

`订单输入` sheet 必须包含以下字段：

| 字段名 | 是否必填 | 含义 | 示例 |
|---|---|---|---|
| 订单 | 是 | 订单名称 | 意诚 |
| 需求量 | 是 | 订单总需求量 | 5700000 |
| 最早开工 | 是 | 订单允许开始生产的最早日期 | 2026/5/1 |
| 最晚完工 | 是 | 订单要求完成的最晚日期 | 2026/5/7 |

示例：

| 订单 | 需求量 | 最早开工 | 最晚完工 |
|---|---:|---|---|
| 意诚 | 5700000 | 2026/5/1 | 2026/5/7 |
| 至上 | 12000000 | 2026/5/1 | 2026/5/15 |

注意事项：

- `需求量` 建议填写整数；
- 日期建议使用 Excel 日期格式，避免输入无法解析的文本；
- `最早开工` 不能晚于 `最晚完工`。

### 2. 插单输入 sheet：`插单输入`

如果 `input_orders.xlsx` 中存在 `插单输入` sheet，且其中存在有效订单，系统会自动进入插单重排模式。

`插单输入` sheet 必须包含以下字段：

| 字段名 | 是否必填 | 含义 | 示例 |
|---|---|---|---|
| 订单 | 是 | 插单订单名称 | 宥阳 |
| 需求量 | 是 | 插单新增需求量 | 2000000 |
| 插单日期 | 是 | 插单发生日期 | 2026/5/10 |
| 最早开工 | 是 | 插单订单允许开始生产的最早日期 | 2026/5/10 |
| 最晚完工 | 是 | 插单订单要求完成的最晚日期 | 2026/5/20 |

插单处理规则：

| 情况 | 系统识别结果 |
|---|---|
| 插单订单名在原订单中不存在 | 新单 |
| 插单订单名在原订单中存在，且旧计划中该订单在插单日期当天或之后仍在生产 | 原订单加量 |
| 插单订单名在原订单中存在，但旧计划中该订单已在插单日前完成 | 同名新批次 |

### 3. 停电计划文件：`停电计划.xlsx`

停电计划文件是可选文件。如果项目根目录下存在 `停电计划.xlsx`，且文件中存在有效停电记录，系统会自动启用停电增强排产模式；如果文件不存在或没有有效记录，则按普通排产逻辑运行。

停电计划文件建议包含以下字段：

| 字段名 | 是否必填 | 含义 | 示例 |
|---|---|---|---|
| 停电开始日期 | 是 | 停电开始日期 | 2026/5/8 |
| 停电结束日期 | 是 | 停电结束日期 | 2026/5/10 |
| 影响产线 | 是 | 受影响产线 | 全部 / 1,2,3 / 1-5 |
| 停电前一天产能比例 | 否 | 停电前一天产能下降比例 | 0.8 / 80% / 80 |

`影响产线` 支持以下写法：

```text
全部
全线
所有
ALL
1,2,3
1，2，3
1、2、3
1-5
1~5
```

如果 `停电前一天产能比例` 为空，程序会在配置范围内随机生成一个产能比例。

## 五、运行方式

在项目根目录下运行：

```bash
python main.py
```

程序会自动执行以下判断：

1. 读取 `input_orders.xlsx` 中的 `订单输入` sheet；
2. 检查是否存在有效的 `插单输入` sheet；
3. 如果没有有效插单，执行普通排产或停电排产；
4. 如果检测到有效插单，执行插单重排；
5. 检查是否存在有效的 `停电计划.xlsx`；
6. 如果存在停电计划，在排产模型中同步考虑停电影响；
7. 求解完成后导出 Excel 排产结果。

## 六、普通排产流程

普通排产流程由 `scheduler_workflow/normal_schedule_flow.py` 负责，主要步骤如下：

1. 读取原始订单数据；
2. 将订单日期转换为模型内部的日期索引；
3. 读取停电计划；
4. 根据订单日期和停电影响确定排产周期；
5. 构建 CP-SAT 决策变量；
6. 添加生产量、产线分配、交期、连续生产和换线等约束；
7. 构建目标函数；
8. 调用 CP-SAT 求解器；
9. 解析求解结果；
10. 导出 Excel 排产结果。

当不存在有效停电计划时，系统执行普通排产。普通排产以订单交期为主要硬约束，在满足订单需求的前提下尽量减少换线、生产跨度、负载波动和超产。

## 七、停电排产逻辑

停电排产主要由 `scheduler_io/outage_loader.py` 和 `preprocessing/capacity_builder.py` 实现。

系统会根据停电计划构建“产线 × 日期”的产能矩阵：

| 日期类型 | 产能处理方式 |
|---|---|
| 正常生产日 | 产线产能为 `DAILY_CAPACITY` |
| 停电前一天 | 产能按停电前一天产能比例下降 |
| 停电期间 | 受影响产线产能为 0 |
| 停电结束后 | 根据恢复比例逐步恢复产能 |
| 全厂停电 | 所有产线当天产能为 0 |

停电相关配置位于 `config.py`，核心配置如下：

```text
POWER_OUTAGE_EXCEL_FILE = "停电计划.xlsx"
PRE_OUTAGE_RATIO_MIN = 0.55
PRE_OUTAGE_RATIO_MAX = 0.85
RANDOM_SEED = 42
RECOVERY_RATIOS = [0.40, 0.80, 0.90, 0.95, 1.00]
ENABLE_SOFT_DUE_FOR_OUTAGE = True
MAX_AUTO_EXTEND_MONTHS_FOR_OUTAGE = 3
```

当 `ENABLE_SOFT_DUE_FOR_OUTAGE` 为 `True` 时，如果停电影响导致当前月份无法完成订单，系统会允许订单延期补产，并通过目标函数惩罚延期订单数量和延期天数。

## 八、插单重排流程

插单重排流程由 `scheduler_workflow/insert_schedule_flow.py` 负责，主要步骤如下：

1. 读取原始订单和插单输入；
2. 读取旧排产结果文件，默认使用 `CP_SAT_排产结果.xlsx`；
3. 从旧排产结果中解析原产线日历；
4. 根据旧计划识别原订单的实际生产状态；
5. 根据插单日期冻结已执行或已下发的旧计划；
6. 自动判断插单类型；
7. 构建插单模式下的 CP-SAT 模型；
8. 在满足插单需求的同时减少原计划扰动；
9. 必要时允许旧订单延期，但对延期订单数量和延期天数进行惩罚；
10. 自动跨月扩展求解，避免因当前月份产能不足导致直接无解；
11. 导出插单排产结果。

插单模式下，旧排产结果支持以下排产图格式：

```text
表2_产线日历
5月排产图
6月排产图
7月排产图
```

插单相关配置位于 `config.py`，核心配置如下：

```text
OUTPUT_EXCEL_FILE = "CP_SAT_排产结果.xlsx"
PREVIOUS_PLAN_EXCEL_FILE = OUTPUT_EXCEL_FILE
INSERT_OUTPUT_EXCEL_FILE = "CP_SAT_插单排产结果.xlsx"
FREEZE_DAYS_AFTER_INSERT = 0
AUTO_EXTEND_MONTHS_FOR_INSERT = 1
MAX_AUTO_EXTEND_MONTHS_FOR_INSERT = 3
QUANTITY_INCREASE_LOOKBACK_DAYS = 3
```

## 十、核心模型说明

本项目将生产排产问题建模为 CP-SAT 约束规划问题。模型通过布尔变量和整数变量描述订单、产线和日期之间的分配关系，并通过硬约束保证排产方案可执行，通过目标函数引导模型生成更符合生产管理需求的方案。

### 1. 主要决策变量

| 变量含义 | 说明 |
|---|---|
| 订单在某天是否生产 | 判断订单是否在指定日期处于生产状态 |
| 订单是否分配到某产线某天 | 判断某条产线某天是否生产某个订单 |
| 订单某天使用的产线数量 | 用于计算日产能、生产连续性和负载波动 |
| 换线变量 | 用于统计相邻日期同一产线上的订单切换 |
| 延期变量 | 用于判断订单是否超过原最晚完工日期 |
| 插单扰动变量 | 用于衡量插单后新计划相对旧计划的变化 |

### 2. 主要硬约束

- 每条产线同一天最多生产一个订单；
- 每个订单只能在允许的生产窗口内生产；
- 订单总产能需要覆盖订单需求量；
- 订单只要当天生产，至少需要占用指定数量的产线；
- 普通模式下尽量保持订单连续生产；
- 停电期间受影响产线不可用；
- 插单模式下冻结插单日期之前的旧计划；
- 原订单加量时，优先保持原产线继续生产该订单。

### 3. 主要优化目标

模型目标函数综合考虑以下因素：

- 减少换线次数；
- 减少订单生产跨度；
- 减少订单每日占线数波动；
- 减少每日总开线数波动；
- 减少超产；
- 停电场景下减少生产日数量和延期；
- 插单场景下减少原计划扰动；
- 插单场景下优先保持加量订单沿原产线继续生产；
- 插单场景下减少延期订单数量和延期天数；
- 插单场景下减少订单被打断形成多段生产；
- 插单场景下提高产线利用率。

## 十一、主要配置参数

核心参数均位于 `config.py`。

### 1. 产线参数

```text
NUM_LINES = 18
DAILY_CAPACITY = 230000
MIN_LINES_PER_ACTIVE_ORDER = 3
```

| 参数 | 含义 |
|---|---|
| NUM_LINES | 产线数量 |
| DAILY_CAPACITY | 单条产线日产能 |
| MIN_LINES_PER_ACTIVE_ORDER | 订单当天只要生产，至少需要分配的产线数量 |

### 2. 求解器参数

```text
MAX_SOLVE_TIME_SECONDS = 300
NUM_SEARCH_WORKERS = 8
```

| 参数 | 含义 |
|---|---|
| MAX_SOLVE_TIME_SECONDS | 单次求解时间上限，单位为秒 |
| NUM_SEARCH_WORKERS | CP-SAT 并行搜索线程数 |

### 3. 目标函数权重

```text
WEIGHT_CHANGEOVER = 1500
WEIGHT_ACTIVE_DAYS = 10
```

## 十三、常见问题与排查

| 问题 | 可能原因 | 处理方式 |
|---|---|---|
| 找不到 `input_orders.xlsx` | 没有在项目根目录运行程序，或文件名被修改 | 确认当前运行目录和文件名 |
| 提示缺少必要列 | Excel 表头名称不一致 | 检查 `订单`、`需求量`、`最早开工`、`最晚完工` 等字段 |
| 日期无法解析 | 日期使用了不规范文本格式 | 建议使用 Excel 日期格式 |
| 插单模式读取旧计划失败 | 没有先生成普通排产结果 | 先运行普通排产，生成 `CP_SAT_排产结果.xlsx` |
| 求解无可行解 | 订单需求过大、交期过紧、停电影响过强或冻结范围过大 | 延长排产窗口、检查停电计划、调整自动跨月扩展参数 |
| 求解时间较长 | 订单数量较多或约束较复杂 | 增加求解时间上限，或适当简化输入规模 |
| 输出结果跨月 | 当前月份产能不足或启用了自动跨月扩展 | 属于正常现象，可查看对应月份排产图 |

## 十五、项目特点

本项目具有以下特点：

1. 使用 CP-SAT 约束规划方法描述多产线订单排产问题；
2. 支持普通排产、停电排产和插单重排三类典型生产场景；
3. 将数据读取、预处理、模型构建、求解、结果解析和 Excel 导出进行模块化拆分；
4. 停电场景中考虑产能下降、停电不可用和停电后逐步恢复；
5. 插单场景中考虑旧计划冻结、原计划扰动、加量订单延续、延期惩罚和自动跨月扩展；
6. 输出结果以 Excel 形式展示，便于生产计划人员查看和后续调整。
